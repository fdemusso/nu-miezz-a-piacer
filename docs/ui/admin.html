★

TopNav & Riepilogo per il team

3 schermate · 5 slice admin coperte · login condiviso (no-slice) · web-only

Top-nav — 4 voci (3 tab + overflow)

  • Dashboard → /admin/dashboard (UsageFrequencyReport + HighDensityZoneMap)
  • Zone → /admin/zones (DefineSensitiveZone + MarkUrbanWarningZone)
  • Report → /admin/reports (MobilityPeriodicReport)
  • Altro → overlay: Profilo ente, Logout

Heatmap densità (AP.04) e KPI vivono dentro la Dashboard, non come pagina autonoma. Allerta urbana (AP.03) è un tipo di zona dentro Gestione zone.

Slice → pagina

  • UsageFrequencyReport (AP.01) → Dashboard + Report
  • MobilityPeriodicReport (AP.02) → Report
  • HighDensityZoneMap (AP.04) → Dashboard (heatmap)
  • DefineSensitiveZone (AP.05) → Zone
  • MarkUrbanWarningZone (AP.03) → Zone

Componenti shadcn da installare

cardbadgebuttontabstabledialogalert-dialogforminputselectlabelpopovercalendarscroll-areaskeletonsonner

Librerie esterne: recharts (grafici), react-leaflet + leaflet.heat + leaflet-draw (heatmap, poligoni zone).

Decisioni aperte

  • Libreria grafici: recharts proposto — confermare vs alternative (visx, chart.js).
  • Export PDF: lato server (qualità) vs client jspdf (semplicità).
  • ZoneType: allineare enum dominio (parking/forbidden/sensitive/incentive) con i tipi UI (ZTL/Pedonale/Scuola/Ospedale/Cantiere).
  • TopNav role-aware: estendere BottomNav.tsx con variante admin da getVisibleSlices('admin').
